### P1-4 resolution 映射（720p/1080p vs 1K/2K/4K）

| 项目 | 路径 | 现有逻辑摘要 | 需修改位置 |
|------|------|--------------|------------|
| 前端选项 | `frontend/src/config/toolConfigs.ts` | `resolutions` 含 `720p`、`1080p`、`2k`、`4k` | 需增加 `1K` 或建立映射表 |
| 前端传参 | `frontend/src/api/generate.ts` | `body.resolution = { preset: params.resolution.toUpperCase() }`，直接透传 | 需在发送前将 `720p`→`1K`、`1080p`→`1K` 或 `2K` 映射 |
| 后端模型 | `backend/app/models/common.py` | `ResolutionConfig.preset: Literal["1K","2K","4K"]`，**不接受 720p/1080p** | 需在接收层或中间层做映射，或扩展 `preset` 类型 |
| 模板使用 | `backend/.../templates/**/*.j2` | 使用 `resolution.preset or "1K"` | 若后端扩展 preset，模板无需改 |

**实现建议**：

1. **方案 A**：前端 `generate.ts` 中增加映射函数，发送前将 `720p`/`1080p` 转为 `1K`，`2k`/`4k` 转为 `2K`/`4K`。
2. **方案 B**：后端 `common.py` 扩展 `preset` 为 `Literal["720p","1080p","1K","2K","4K"]`，并在 Swiftask 调用前映射到 1K/2K/4K（若 API 仅支持后者）。

**推荐方案 A**，改动集中在前端，后端保持 1K/2K/4K 语义。

## 五、数据流说明

### 双图模式（banana-pro-dual、style-mimic）

1. **前端**：`ToolDetail` 渲染两个 UploadZone，分别通过 `handleFileSelect`、`handleSecondFileSelect` 设置 `imageBase64`、`secondImageBase64`。
2. **generate.ts**：将 `imageBase64`、`secondImageBase64` 依次 push 到 `body.images`。
3. **后端**：`GenerationRequest.images` 接收 `[{base64_data, format}, {base64_data, format}]`，顺序为：`images[0]=目标图`，`images[1]=风格参考`（与 unified 约定一致）。

### resolution 映射建议

```ts
// generate.ts 中
const RESOLUTION_MAP: Record<string, string> = {
  '720p': '1K',
  '1080p': '1K',
  '2k': '2K',
  '4k': '4K',
};
const preset = params.resolution && params.resolution !== 'default'
  ? (RESOLUTION_MAP[params.resolution.toLowerCase()] ?? params.resolution.toUpperCase())
  : null;
if (preset) body.resolution = { preset };
```
